Software Technology Projects

iou
The iou project is an open source toolbox full of server side structures and helpers. The most current version is in offline development.
apd
The apd server (1998/9) was a small, resource oriented HTTP application server. Similar sense but different to the W3C JigSaw. Resources are I/O stream devices like files and sockets. Apd excelled at serving static content. Apd included a simple and effective application programming structure.
rhinojetty
The rhinojetty project is an open source HTTP Server implemented in JavaScript over Java (Mozilla Rhino). It includes "RJ" JavaScript Server Pages, a unique dialect of JSP and JavaScript Classes and Functions. Rhinojetty includes a fast compiled page product engine -- changed pages are compiled to interpreted output byte stream binary for extreme responsiveness.
sx
SX is the current HTTP server software for syntelos.com.
openfoam
Open foam is a name for OAEP padding under RC (4). The combination is a code level experiment in considering alternative key stream generators for RC with this padding.

This experiment seeks answers to two questions. What are the requirements for a key stream generator over this padding? And, is there a provably secure key stream generator for this combination?

The openfoam code is offline.
edo6
Edo6 is an interpreted server side programming language designed for extreme flexibility in the use, construction and personalization of its applications. Business analysts manage an application system on behalf of groups of users in a production environment. Application construction, management and reconfiguration features.
jing
The jing project is an incomplete (open) source base for SWF (Adobe Flash binaries) assembly and disassembly. It provides the initial grunt work for an SWF code base, but not an implementation.

Research & Development

Interaction
Interaction is a subtle subject with many aspects for internet application engineering.  Interface architecture solves all problems by focusing definition, solution, design and development on user and network interfaces. 
Representational State Programming
Scalability for high volume applications in constructive transactions.  RESP extends Roy Fielding's well known Representational State Transfer (REST) architectural style for transactional and constructive web applications.  RESP describes ongoing research and development into the application of the principles of Fielding's REST to the dynamic programming of interactive applications.
Interface Architecture
A unified architectural methodology for the design and development of correct, robust and scalable interactive software systems.  Interface Architecture reiterates the well known discipline for the specification of network services, and applies these concepts to the principal components of an interactive system including the Graphical User Interface, Web Services and Host APIs.  It is simple, effective, and scales to large and distributed development groups.
