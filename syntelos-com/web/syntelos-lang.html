Description

The SPL has been conceived as a data code language for programming by type and set metaphors. A data code language is intended as maximizing the value of code as data through a regular grammar and a document object model parsing library. Programming with type reasoning is well known as strongly typed programming languages. Programming with set reasoning is intended as working with streams and iterators in terms of producing their subsets from the operational declaration of a function or method.

The well known problem of software bugs derived from common code patterns can be solved at the language level. The approach minimizes the occurrence of familiar issues and maximizes productivity. The SPL has been conceived with operators and facilities for deriving logical sets from streams and iterators, and performing common array management operations, as part of a larger solution to the programming language problem.

With the envisioned approach a programming language may position operational programming as the declaration of semantics. The effort is to remove the operational programming patterns that detract from the declaration of semantics, and reasoning about software systems.
